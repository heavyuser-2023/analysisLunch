analysisLunch · 점심봇 인스타그램 연동

인스타그램 API 토큰 두 값,
처음부터 끝까지 만들기

텔레그램과 달리 인스타그램은 Meta(페이스북) 설정이 필요합니다. 이 가이드를 위에서부터 순서대로 따라가면 아래 두 값을 만들 수 있습니다. 한 번만 만들면 됩니다.

INSTAGRAM_ACCESS_TOKEN
인스타에 글을 올릴 권한이 담긴 열쇠. 시스템 사용자 토큰으로 만들면 만료 없음.
INSTAGRAM_BUSINESS_ACCOUNT_ID
글을 올릴 대상 인스타 비즈니스 계정의 숫자 ID.
준비

계정부터 갖추기

스마트폰 + 브라우저

먼저 이 3가지가 연결돼 있어야 토큰을 만들 수 있습니다. 이미 돼 있으면 다음 단계로 넘어가세요.

  1. 인스타 계정을 “프로페셔널”로 전환

    개인 계정으론 API 게시가 안 됩니다. 비즈니스/크리에이터로 바꿔야 합니다.

    • 인스타그램 앱 → 내 프로필 → 우측 상단 ☰ 메뉴 → 설정 및 개인정보
    • 계정 유형 및 도구 → 프로페셔널 계정으로 전환
    • 비즈니스 선택 (크리에이터도 가능)

  2. 페이스북 페이지 만들기 (없으면)

    인스타 API는 반드시 페이스북 페이지를 거칩니다. 개인 타임라인 말고 “페이지”입니다.

    • 브라우저에서 facebook.com/pages/create 접속
    • 페이지 이름(예: 쉐프스케치 점심)만 넣고 생성하면 끝

  3. 인스타 ↔ 페이스북 페이지 연결

    • 인스타 앱 → 프로필 → 프로필 편집 → 페이지 항목
    • 방금 만든 페이스북 페이지를 선택해 연결
    완료 기준 “인스타 비즈니스 계정”이 “페이스북 페이지”에 묶였습니다. 이게 모든 것의 토대입니다.
A

Meta 앱 & 비즈니스 설정

developers.facebook.com

  1. Meta 개발자 앱 만들기

    • developers.facebook.com 접속 → 우측 상단 내 앱 → 앱 만들기
    • 유형은 “비즈니스(Business)” 선택 → 앱 이름 입력 → 생성
    • 앱 대시보드에서 제품 추가 → Instagram Graph API 추가

  2. 비즈니스 관리자에 자산 연결

    business.facebook.com → 비즈니스 설정에서 세 가지를 같은 비즈니스 안에 모읍니다.

    • 계정 → 페이지: 만든 페이스북 페이지 추가
    • 계정 → 앱: 방금 만든 앱 추가
    • 계정 → Instagram 계정: 인스타 비즈니스 계정 추가
    왜 필요한가 토큰이 “페이지 → 인스타 계정”에 접근하려면, 세 자산이 한 비즈니스 안에 있어야 권한을 줄 수 있습니다.
B

만료 없는 토큰 만들기

→ INSTAGRAM_ACCESS_TOKEN

일반 사용자 토큰은 60일이면 끊깁니다. 자동화엔 시스템 사용자 토큰(만료 없음)을 씁니다.

  1. 시스템 사용자 생성

    • 비즈니스 설정 → 사용자 → 시스템 사용자 → 추가
    • 이름 입력 (예: lunch-bot) → 역할 관리자 → 생성

  2. 시스템 사용자에게 자산 할당

    만든 시스템 사용자를 클릭 → 자산 할당:

    • 페이지 선택 → 권한 “전체 제어(콘텐츠 관리)” 켜기
    • 앱도 같은 방식으로 할당

  3. 토큰 생성 — 핵심 단계

    같은 화면에서 토큰 생성 클릭:

    • 앱 선택
    • 토큰 만료: “안 함(Never)” ← 무기한의 핵심
    • 권한 체크: instagram_basic instagram_content_publish pages_show_list pages_read_engagement business_management
    • 생성된 긴 토큰 문자열 복사 → 이게 INSTAGRAM_ACCESS_TOKEN
    딱 한 번만 보임 토큰은 생성 직후에만 전체가 표시됩니다. 바로 복사해 안전한 곳에 보관하세요. 잃어버리면 다시 생성해야 합니다.
C

비즈니스 계정 ID 구하기

→ INSTAGRAM_BUSINESS_ACCOUNT_ID

  1. 내 페이지 ID 알아내기

    B에서 만든 토큰으로 내가 접근 가능한 페이지 목록과 ID를 조회합니다. 터미널에 붙여넣으세요.

    # <TOKEN> 자리에 B에서 만든 토큰을 넣으세요
curl -s "https://graph.facebook.com/v21.0/me/accounts?access_token=<TOKEN>"

    응답의 "id" 가 페이지 ID입니다 (페이지가 여러 개면 이름으로 구분).

  2. 페이지에 연결된 인스타 계정 ID 조회

    위에서 얻은 페이지 ID로 인스타 비즈니스 계정 ID를 꺼냅니다.

    curl -s "https://graph.facebook.com/v21.0/<페이지ID>?fields=instagram_business_account&access_token=<TOKEN>"

    응답 예시 — instagram_business_account.id 가 바로 그 값입니다.

    { "instagram_business_account": { "id": "17841400000000000" },
  "id": "1020..." }
    획득 그 숫자(17841…)가 INSTAGRAM_BUSINESS_ACCOUNT_ID 입니다.
D

값 넣고 검증하기

프로젝트 적용

  1. 로컬 테스트용 .secrets 에 추가

    INSTAGRAM_ACCESS_TOKEN=EAAG...(B에서 복사한 토큰)
INSTAGRAM_BUSINESS_ACCOUNT_ID=17841400000000000

  2. 운영(GitHub Actions) 시크릿에 등록

    gh secret set INSTAGRAM_ACCESS_TOKEN --body "EAAG..."
gh secret set INSTAGRAM_BUSINESS_ACCOUNT_ID --body "17841400000000000"

    워크플로우는 이미 이 두 시크릿을 읽도록 연결돼 있습니다.

  3. 토큰이 정말 “만료 없음”인지 검증

    아래로 토큰 상태를 확인합니다. expires_at 이 0 이면 무기한입니다.

    curl -s "https://graph.facebook.com/debug_token?input_token=<TOKEN>&access_token=<TOKEN>"
    읽는 법 "expires_at": 0 → 만료 없음 ✓ · "is_valid": true → 유효 ✓ · scopes 에 instagram_content_publish 포함 확인.
무기한이어도 깨질 수 있는 경우 토큰 만든 관리자 계정의 비밀번호 변경, 앱 시크릿 재설정, 권한·자산 회수, 시스템 사용자 삭제 시엔 무효화됩니다. 시스템 사용자/관리자 계정을 따로 안정적으로 관리하세요.

코드는 토큰 종류와 무관하게 두 환경변수를 그대로 사용합니다 — 값만 채우면 동작합니다. 설정 누락 시 인스타그램 단계는 조용히 건너뜁니다(다른 채널 영향 없음). 버전 표기 v21.0 은 필요 시 최신으로 바꿔도 됩니다.
